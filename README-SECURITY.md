# 🔒 セキュリティアーキテクチャ補足

## API キーの安全な管理

このプロジェクトは **セキュアなバックエンドプロキシ** を実装しており、API キーはフロントエンドに一切露出しません。

### 🔐 セキュアモード（推奨・本番環境用）

**アーキテクチャ図**:
```
ブラウザ → Vercel Functions (API Proxy) → Zhipu AI API
         (APIキーなし)    (環境変数で安全に管理)
```

**特徴**:
- ✅ API キーはサーバー側の環境変数にのみ存在
- ✅ フロントエンドコードには一切含まれない
- ✅ DevTools で API キーを見ることができない
- ✅ 不正利用のリスクが大幅に低減

**使用方法**:
1. Vercel で `ZHIPU_API_KEY` 環境変数を設定
2. `src/api-secure.js` を使用（デフォルト）
3. デプロイ

**ファイル**:
- `api/chat.js` - バックエンド API プロキシ
- `src/api-secure.js` - セキュアなフロントエンド API クライアント
- `build-secure.sh` - セキュアビルドスクリプト

詳細は [SECURITY.md](SECURITY.md) を参照してください。

### ⚠️ レガシーモード（開発・テスト用のみ）

**⚠️ 警告**: このモードは API キーがフロントエンドに露出します。本番環境では使用しないでください。

フロントエンドから直接 API を呼び出す従来の方法も、ローカル開発用に残されています：

**ファイル**:
- `src/api.js` - 直接 API 呼び出し（非推奨）
- `build.sh` - 従来のビルドスクリプト

**リスク**:
- 🔴 API キーがブラウザの DevTools で見える
- 🔴 不正利用される可能性が高い
- 🔴 本番環境では絶対に使用しないこと

## セキュリティ比較

| モード | API キー保存場所 | DevTools可視性 | 本番環境使用 |
|--------|-----------------|----------------|-------------|
| **セキュアモード** | サーバー環境変数 | ❌ 見えない | ✅ 推奨 |
| レガシーモード | フロントエンド | ✅ 見える | ❌ 禁止 |

## デプロイ前チェックリスト

本番環境にデプロイする前に、以下を確認してください：

- [ ] Vercel で `ZHIPU_API_KEY` 環境変数を設定
- [ ] `vercel.json` で `buildCommand: "bash build-secure.sh"` を使用
- [ ] フロントエンドコードで `api-secure.js` をインポート
- [ ] `config.js` に API キーが含まれていないことを確認
- [ ] デプロイ後、DevTools で API キーが見えないことを確認
